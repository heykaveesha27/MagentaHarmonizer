1Install ASIO4ALL

If you haven't already installed ASIO4ALL:

  1. Download ASIO4ALL from: https://www.asio4all.org
  2. Run the installer and follow the setup wizard
  3. Restart your computer after installation

2Configure ASIO4ALL Control Panel

  1. Open the ASIO4ALL Control Panel:
    • Search for "ASIO4ALL" in Windows Start Menu, OR
    • Go to: C:\Program Files (x86)\ASIO4ALL v2\ and run ASIO4ALL v2 Settings.exe
  2. In the ASIO4ALL panel:
    • Enable your audio device (check the box next to it)
    • Set buffer size:
      • Lower = less latency but may cause crackling
      • Recommended: 256-512 samples
  3. Click Apply and close the panel

3Set ASIO4ALL as Windows Default (Optional)

  1. Right-click the speaker icon in your Windows taskbar
  2. Select "Sound settings"
  3. Under "Advanced sound options", click "App volume and device preferences"
  4. Find your browser (Chrome, Edge, Firefox) in the list
  5. Set Output to your ASIO4ALL-enabled device
üí° Note: ASIO drivers typically work best with DAWs (Digital Audio Workstations) and desktop applications. For web browsers, you may need to use the Windows audio device that ASIO4ALL is routing through.

4Alternative: Use a Dedicated DAW or Audio Host

For true ASIO low-latency support, consider using:

  • REAPER (free evaluation) - Can host VST plugins and route MIDI
  • VSTHost (free) - Simple ASIO host application
  • FL Studio, Ableton Live, etc. - Full-featured DAWs with ASIO support

These applications can directly access ASIO4ALL for minimal latency.

5Optimize Browser Audio Settings

  1. Close all browser windows completely
  2. Ensure ASIO4ALL device is set as default in Windows Sound settings
  3. Reopen your browser
  4. Navigate to the Music Generator application
  5. When prompted, allow audio access
‚úÖ Testing Your Setup:
  1. Connect your MIDI keyboard
  2. Launch the Music Generator
  3. Check if the MIDI device name appears
  4. Play a note and listen for latency
  5. If latency is high, reduce ASIO4ALL buffer size and restart

6Troubleshooting

  • No sound: Make sure ASIO4ALL device is enabled in the control panel
  • Crackling/glitches: Increase ASIO4ALL buffer size
  • High latency: Decrease ASIO4ALL buffer size (try 128 or 256 samples)
  • Browser not using ASIO: Check Windows Sound settings ‚Üí App-specific settings
  • MIDI not working: Ensure MIDI keyboard is connected before opening browser
